# 구현 계획 - 워크스테이션 전략 프레젠테이션 현대화

기존 `index.html`을 세련되고 현대적인, 첨단 기술 느낌의 프레젠테이션으로 변환하는 것이 목표입니다. 전략 브리핑에 적합한 시각적 효과, 가독성, 그리고 "프리미엄" 느낌에 중점을 둡니다.

## 사용자 검토 필요 사항

> [!IMPORTANT]
> 모범 사례를 따르기 위해 내부 CSS와 JS를 별도 파일(`style.css`, `script.js`)로 분리할 예정입니다.
> 디자인 방향은 "하이테크/의료" 분위기를 강조하기 위해 **다크 모드** 미학(딥 네이비/블랙 배경에 시안/블루 강조색)으로 전환됩니다.

## 제안된 변경 사항

### [프레젠테이션 로직 및 구조]
#### [수정] [index.html](index.html)
- 내부 스타일 및 스크립트 제거.
- `style.css` 및 `script.js` 연결.
- 새로운 레이아웃 지원을 위한 HTML 구조 업데이트 (예: 전체 화면 배경, 더 나은 그리드 레이아웃).
- 구글 폰트 링크 추가 (Inter/Roboto 등).

#### [신규] [style.css](style.css)
- 다크 모드 테마 구현.
- 색상, 간격, 폰트를 위한 변수 설정.
- 모던 CSS 리셋.
- 슬라이드를 위한 Flexbox/Grid 레이아웃.
- 애니메이션: 슬라이드 전환 (페이드/슬라이드), 요소 순차 등장.
- 스타일링된 컴포넌트: 글래스모피즘(Glassmorphism) 카드, 빛나는 테두리, 모던한 표/차트.

#### [신규] [script.js](script.js)
- 슬라이드 내비게이션 로직 재구현.
- 키보드 지원 추가 (화살표 키).
- 간단한 애니메이션 트리거 추가 (Vanilla JS 또는 CSS 클래스를 활용한 GSAP 유사 효과).
- ROI 차트 애니메이션 로직.

### [에셋]
- 플레이스홀더를 대체할 배경 및 컨셉 이미지 생성.

## 검증 계획

### 자동화 테스트
- 없음 (시각적 스타일링 작업).

### 수동 검증
1. Chrome에서 `index.html` 열기.
2. 모든 슬라이드가 새로운 디자인으로 올바르게 렌더링되는지 확인.
3. 내비게이션 확인 (이전/다음 버튼, 키보드 화살표).
4. 애니메이션이 부드럽게 재생되는지 확인.
5. 반응형 확인 (다양한 화면 크기에서 올바르게 조정되거나 중앙에 배치되는지 확인, 주로 데스크탑 프레젠테이션용이지만).
